# 碎石记（Crusher）后台管理使用指导手册

## 一、后台登录

1. 使用管理员账号登录平台
2. 访问 `/admin` 路径进入管理后台
3. 系统自动验证用户角色，非管理员账号无法访问后台

---

## 二、用户管理

### 2.1 用户列表

访问路径：`/admin`（后台首页即用户管理）

- 查看所有注册用户：ID、邮箱、用户名、注册时间、当前积分、账号状态
- 支持按邮箱、用户名搜索
- 支持按账号状态筛选：
  - **normal**：正常
  - **login_disabled**：禁止登录
  - **usage_disabled**：禁止使用功能
- 支持按积分范围筛选

### 2.2 用户详情

点击用户进入详情页（`/admin/users/[id]`）：

- 查看用户基本信息和积分余额
- 查看待审批的积分操作
- 查看已完成的积分操作历史
- 执行以下操作：

| 操作 | 说明 |
|------|------|
| **手动赠送积分** | 为用户增加积分，需填写操作说明，提交后进入审批流程 |
| **手动扣减积分** | 从用户账户扣除积分，需填写操作说明，提交后进入审批流程 |
| **禁止登录** | 封禁违规用户，用户无法登录平台 |
| **禁止使用** | 允许登录但禁止使用分析等核心功能 |
| **恢复正常** | 解除禁用状态，恢复用户正常使用 |

> **注意**：积分手动调整需要其他管理员审批，操作发起人不能审批自己提交的申请。

---

## 三、订单管理

访问路径：`/admin/orders`

### 3.1 订单列表

- 查看所有支付订单：订单号、用户信息、套餐类型、金额（分）、积分数量、支付时间
- 支持按订单状态筛选：
  - **pending**：待支付
  - **paid**：已支付
  - **failed**：支付失败
  - **refunded**：已退款

### 3.2 退款处理

1. 在订单列表找到目标订单，点击"退款"
2. 填写退款原因，提交退款申请
3. 退款申请进入审批流程，由其他管理员审批
4. 审批通过后，系统自动扣除用户对应积分并标记订单为已退款

---

## 四、用量统计

访问路径：`/admin/stats`

统计维度（支持按日查看）：

- **新增用户数**：每日注册用户数量
- **订单数**：每日支付订单数量
- **收入**：每日支付金额（元）
- **积分消耗**：每日用户消耗的积分总量
- **活跃用户数**：每日有操作行为的用户数量

数据以图表形式展示（柱状图、折线图）。

---

## 五、反馈管理

访问路径：`/admin/feedbacks`

### 5.1 反馈概览

页面顶部展示统计卡片：
- 今日新增反馈数
- 待处理反馈数
- 已解决反馈数

### 5.2 反馈列表

- 查看所有用户反馈：类型、标题、内容、提交用户、状态、提交时间
- 支持按反馈类型筛选：支付相关、Bug、功能建议、其他
- 支持按状态筛选：待处理、处理中、已解决、已关闭

### 5.3 处理反馈

1. 点击"处理"打开处理对话框
2. 更新反馈状态（待处理 → 处理中 → 已解决 / 已关闭）
3. 填写管理员备注，记录处理情况
4. 提交后状态实时更新

---

## 六、系统公告管理

访问路径：`/admin/announcements`

### 6.1 发布公告

1. 点击"创建公告"
2. 填写公告标题和内容（支持 Markdown 格式）
3. 选择公告类型：
   - **info**（信息）：普通通知
   - **warning**（警告）：重要提醒
   - **maintenance**（维护）：系统维护通知
4. 设置过期时间（可选，不设置则长期有效）
5. 发布后公告在用户首页顶部横幅展示

### 6.2 管理公告

- **编辑**：修改公告标题、内容、类型、过期时间
- **停用/启用**：控制公告是否对用户可见
- **删除**：永久删除公告

---

## 七、Prompt 版本管理

访问路径：`/admin/prompts`

### 7.1 查看版本历史

1. 选择透镜类型（需求分析、会议纪要、文档评审等）
2. 查看该透镜的所有 Prompt 版本列表
3. 当前激活版本标记为"活跃"

### 7.2 创建新版本

1. 点击"创建新版本"
2. 输入新的 Prompt 内容
3. 填写变更说明（记录本次修改的目的）
4. 提交后新版本处于未激活状态

### 7.3 激活版本

- 点击"激活"将指定版本设为当前使用版本
- 激活后，用户的新分析请求将使用该版本的 Prompt
- 支持随时切换回历史版本（版本回滚）

### 7.4 测试 Prompt

1. 在版本详情页点击"测试"
2. 输入测试文本内容
3. 点击"运行测试"，实时预览 AI 输出结果
4. 确认效果满意后再激活该版本

---

## 八、积分审批

积分手动调整（赠送/扣减）和退款申请均需经过审批流程：

1. 操作发起人提交申请后，申请进入待审批队列
2. 其他管理员在用户详情页或订单页查看待审批项
3. 审批通过：系统自动执行积分变更，记录交易流水
4. 审批拒绝：操作取消，用户积分不变
5. **限制**：操作发起人不能审批自己提交的申请

---

## 九、安全注意事项

1. 管理员账号请妥善保管，定期更换密码
2. 所有后台操作均有操作日志记录，请勿进行违规操作
3. 处理用户数据时请遵守隐私保护法规，不得泄露用户个人信息和文档内容
4. 系统升级或维护前请提前发布公告，告知用户维护时间和影响范围
5. 积分调整和退款操作需谨慎，审批前请核实用户信息和操作原因
