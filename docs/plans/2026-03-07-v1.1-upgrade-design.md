## 一、升级背景

v1.0 完成后，发现以下问题需要在 v1.1 中修复和增强：

1. 首页与文档模块内容重复，缺乏数据价值
2. 透镜分析只能单文档操作，场景受限
3. 分析历史入口分散，不易找到
4. Admin 模块有未实现入口（系统设置）和功能 Bug（积分调整 API 崩溃）
5. 404 / 错误页面使用 Next.js 默认样式，体验差

### 功能 2：多文档合并透镜分析

**涉及文件**：
- `src/app/(main)/documents/page.tsx`（或其内部组件）
- `src/components/documents/document-list.tsx`（文档列表组件）
- `src/app/api/insights/route.ts`（AI 分析 API）
- 数据库 migration（`insights` 表新增字段）

#### 2.1 数据库变更

```sql
-- insights 表新增多文档关联字段
ALTER TABLE insights ADD COLUMN document_ids UUID[] DEFAULT NULL;
```

> 单文档分析时 `document_ids` 为 `NULL`，多文档分析时填充选中的文档 ID 数组。

#### 2.2 前端交互设计

**文档列表页**增加多选模式：

```
[普通模式] ←→ [多选模式]（右上角切换按钮）

多选模式下：
- 每个文档卡片左侧显示复选框
- 选中后卡片高亮
- 底部出现悬浮操作栏：

┌─────────────────────────────────────────────────────┐
│  已选 3 篇 · 合并字数 4,280 字 · 预计消耗 22 积分      │
│                                    [开始分析透镜] →  │
└─────────────────────────────────────────────────────┘
```

**选择透镜**：点击「开始分析透镜」后，弹出透镜选择 Sheet（与单文档分析保持一致的 UI）。

#### 2.3 后端处理逻辑

```
POST /api/insights
{
  document_ids: ["uuid1", "uuid2", "uuid3"],  // 多文档
  lens_type: "requirements",
  ...
}

后端处理：
1. 批量读取文档内容
2. 按选择顺序拼接：
   --- 文档：{标题} ---
   {内容}

   --- 文档：{标题} ---
   {内容}
3. 计算总字数，按现有公式计算积分
4. 调用 AI，结果写入 insights，document_ids 填充数组
```

#### 2.4 兼容性

- 单文档分析路径完全不变（`document_id` 单个 ID，`document_ids` 为 NULL）
- 多文档分析时 `document_id` 可存第一篇文档的 ID（主文档），`document_ids` 存完整数组

### 功能 6：修复管理员积分调整功能

**涉及文件**：
- `src/app/api/admin/credits/route.ts`（完全重写）
- `src/app/admin/users/[id]/admin-credit-form.tsx`（修正 `type` 字段）

#### 6.1 API 重写

**废弃**：`pending_credit_transactions` 相关逻辑（表不存在，彻底移除）。

**新逻辑**：

```
POST /api/admin/credits
{
  userId: "uuid",
  amount: 100,          // 正数=增加，负数=扣减
  type: "manual_grant" | "admin_deduct",
  description: "PayPal 转账 ¥45，订单 ORD-001"
}

处理流程：
1. 验证管理员身份（requireAdmin）
2. 验证目标用户存在（createAdminClient 查 profiles）
3. 读取用户当前余额（profiles.credits）
4. 计算新余额 = 当前余额 + amount
5. 校验：扣减时新余额不能 < 0
6. 事务操作（createAdminClient）：
   a. UPDATE profiles SET credits = 新余额 WHERE id = userId
   b. INSERT INTO credit_transactions (
        user_id, amount, balance_after, type,
        description, operated_by, created_at
      )
7. 返回 { success: true, newBalance }
```

#### 6.2 前端修正

**文件**：`src/app/admin/users/[id]/admin-credit-form.tsx`

修正 `handleSubmit`：
- 补全 `type` 字段（`grant` → `manual_grant`，`deduct` → `admin_deduct`）
- 更新成功提示显示新余额
- 扣减时前端也提示不能超过余额
